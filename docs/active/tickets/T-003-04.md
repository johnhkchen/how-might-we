## Context

Build the Stage 1 (Setup) UI components: PersonaCard and ConstraintList. PersonaCard shows the structured persona with all fields inline-editable. ConstraintList shows constraints with type badges and edit/delete/add capabilities. Both components should support streaming updates (fields filling in progressively).

## Acceptance Criteria

- [ ] PersonaCard displays label, role, goals, frustrations, context, influencers
- [ ] All PersonaCard fields are inline-editable (click to edit, blur to save)
- [ ] Edits update the session store
- [ ] ConstraintList shows constraints with colored type badges: "hard" (red), "soft" (yellow), "assumption" (blue)
- [ ] Constraints are individually editable and deletable
- [ ] "Add constraint" button creates a new constraint
- [ ] Both components handle partial/streaming data gracefully (fields appear as they arrive)
- [ ] Stage 1 section in workshop page: text inputs for persona description, domain, constraints → calls `/api/persona` → shows PersonaCard + ConstraintList
