## Context

Add Cloudflare Turnstile bot protection to prevent automated abuse of the LLM-backed API. The frontend embeds the Turnstile widget, and the CF Worker verifies the token before proxying to Lambda.

## Acceptance Criteria

- [ ] Turnstile widget embedded on the workshop page (invisible or managed mode)
- [ ] Token sent with each API request (e.g., in `X-Turnstile-Token` header)
- [ ] Worker verifies token via Turnstile siteverify API before proxying
- [ ] Invalid/missing tokens return 403
- [ ] Turnstile site key configurable via environment variable
