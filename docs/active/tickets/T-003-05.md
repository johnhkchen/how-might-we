## Context

Build the Stage 2 (Analyze) UI: AnalysisPanel. The user enters their rough HMW statement, and the panel streams in the critique — implicit user, embedded assumptions, scope assessment, solution bias, underlying tension, and initial reframing. The critique should build progressively, reflecting BAML's streaming output.

## Acceptance Criteria

- [ ] Text input for the raw HMW statement
- [ ] "Analyze" button calls `/api/analyze` with statement + ProblemContext from store
- [ ] Analysis streams in section by section: implicit user → assumptions → scope level → solution bias → underlying tension → initial reframing
- [ ] Scope level displayed with visual indicator (too narrow / too broad / well scoped)
- [ ] Solution bias highlighted if present
- [ ] Underlying tension visually emphasized (this is the key insight)
- [ ] Initial reframing shown as a preview of what expansion will produce
- [ ] Analysis stored in session store
