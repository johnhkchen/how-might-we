## Context

The SSE client utility (`stream.ts`) and mock API layer (`mock.ts`, `client.ts`) are scaffolded but need to be verified and potentially refined. Ensure the mock fixtures produce realistic BAML-typed streaming behavior and the `streamFromAPI` function correctly handles partial JSON objects, edge cases, and errors.

## Acceptance Criteria

- [ ] `streamFromAPI` correctly parses SSE events with buffering for split chunks
- [ ] Mock mode (`VITE_MOCK_API=true`) returns fixture data with realistic timing delays
- [ ] Each mock endpoint streams progressively building typed objects (matching BAML streaming behavior)
- [ ] `onPartial` callback fires for each partial with the correct types
- [ ] Error cases handled: non-200 response throws, missing body throws, malformed JSON skipped
- [ ] `npm run dev:mock` demonstrates streaming in browser console
