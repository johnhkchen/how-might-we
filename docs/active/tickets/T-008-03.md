## Context

All 4 BAML functions currently use Claude Sonnet 4.6. Some tasks don't need that level of capability:

- **RefinePersona** — structured extraction from rough text. Haiku handles this well.
- **AnalyzeHMW** — needs nuanced critique. Keep on Sonnet.
- **ExpandHMW** — creative reframing. Keep on Sonnet.
- **RefineHMW** — context-aware creative work. Keep on Sonnet.

Haiku is ~10x cheaper than Sonnet per token. Routing persona refinement to Haiku saves tokens on the most common first call.

## Changes Required

### BAML clients (`clients.baml`)
Add a Haiku client alongside the existing Claude (Sonnet) client:
```baml
client<llm> ClaudeHaiku {
  provider anthropic
  options {
    model "claude-haiku-4-5-20251001"
    api_key env.ANTHROPIC_API_KEY
    max_tokens 1024
  }
}
```

Also reduce `max_tokens` on the Sonnet client from 4096 to 2048 (we don't need that much with the verbosity constraints).

### BAML functions
- `persona.baml` — change `client Claude` to `client ClaudeHaiku`
- `analyze.baml` — keep `client Claude`
- `expand.baml` — keep `client Claude`
- `refine.baml` — keep `client Claude`

### Verify
Test persona refinement with Haiku locally — output quality should be comparable for this structured task.
