## Context

Session state is entirely in-memory — a page refresh loses all progress. For a workshop tool where users invest 10-20 minutes sculpting HMW questions, this is painful. Save session state to localStorage and offer recovery on return.

## Acceptance Criteria

- [ ] Session state auto-saved to localStorage on every meaningful change (debounced, not on every keystroke)
- [ ] On page load, if a saved session exists, show a "Resume previous session?" prompt
- [ ] "Resume" restores full state: persona, context, analysis, candidates with statuses, clipped list, iteration count
- [ ] "Start fresh" clears the saved session
- [ ] Saved session includes a timestamp; sessions older than 24 hours are auto-discarded
- [ ] `Set<string>` (clippedIds) serialized/deserialized correctly (JSON doesn't support Sets natively)
- [ ] Works alongside mock mode
