## Context

When the user hits a rate limit (429 response), the frontend shows a generic "API error: 429" message. There's no retry guidance, countdown, or clear explanation. The `X-RateLimit-Remaining` and `Retry-After` headers from the worker should be surfaced to the user.

## Acceptance Criteria

- [ ] 429 responses show a dedicated rate limit message (not generic error)
- [ ] Message includes how long to wait (from `Retry-After` header)
- [ ] Show remaining request count from `X-RateLimit-Remaining` header (subtle, e.g., tooltip or small text)
- [ ] Buttons are disabled during the retry-after period
- [ ] Countdown timer shown until the user can try again
- [ ] Works for all 4 API endpoints (persona, analyze, expand, refine)
