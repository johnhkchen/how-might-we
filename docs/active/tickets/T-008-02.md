## Context

BAML outputs are too verbose ‚Äî long rationales, multi-paragraph contexts, exhaustive lists. This wastes tokens and creates walls of text in the UI. Constrain all outputs to be concise.

The rule: **3 short lines or 2 medium-length lines max** per field. Apply this across all BAML prompts and type descriptions.

## Changes Required

### All BAML types (`types.baml`)
Add `@description` constraints to chatty fields:
- `Persona.context` ‚Äî "2 sentences max"
- `Persona.goals` / `frustrations` / `influencers` ‚Äî "3-5 items, each under 10 words"
- `HMWVariant.rationale` ‚Äî "1-2 sentences max"
- `HMWAnalysis.underlyingTension` ‚Äî "1-2 sentences"
- `HMWAnalysis.embeddedAssumptions` ‚Äî "3-5 items, each under 15 words"
- `Constraint.challengeRationale` ‚Äî "1 sentence"
- `HMWRefinement.recommendation` ‚Äî "2-3 sentences max"
- `HMWRefinement.tensions` ‚Äî "2-3 items, each 1 sentence"

### All BAML prompts
Add explicit brevity instructions to each prompt:
- "Be concise. Each field should be 1-2 sentences unless it's a list."
- "List items should be short phrases, not paragraphs."
- For persona: "Infer only what the input implies. Don't pad with generic SaaS designer traits."

### Mock fixtures
Update to match shorter expected outputs.

## Acceptance Criteria

- [ ] No single field in any BAML output exceeds ~50 words (except arrays of items)
- [ ] Array fields (goals, frustrations, assumptions) have 3-5 items, each concise
- [ ] Rationale fields are 1-2 sentences, not paragraphs
- [ ] Persona context is 2 sentences, not a full biography
- [ ] Total token output per call reduced by ~40-60% compared to current
- [ ] UI still reads well ‚Äî concise doesn't mean cryptic
- [ ] Mock fixtures updated
