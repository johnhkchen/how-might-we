## Context

If the Turnstile script fails to load (network error, adblocker, etc.), the frontend silently sends requests without a Turnstile token. If the worker enforces token validation, the user gets a 403 with no explanation. There should be clear feedback and a graceful fallback.

## Acceptance Criteria

- [ ] Detect Turnstile script load failure (timeout or error event)
- [ ] Show a non-blocking warning banner if Turnstile can't load
- [ ] If a 403 is returned due to missing/invalid token, show a specific message ("Bot protection check failed — try refreshing")
- [ ] Provide a "Retry verification" button that re-initializes Turnstile
- [ ] Workshop still functions if Turnstile is disabled (no site key configured)
