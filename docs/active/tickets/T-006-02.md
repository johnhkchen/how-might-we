## Context

BAML downloads a 54MB native library (`libbaml_cffi-aarch64-unknown-linux-gnu.so`) on every Lambda cold start, adding ~1-2 seconds to init. The library is cached in `/tmp/.cache/baml/libs/` but `/tmp` is cleared between cold starts.

The fix: download the `.so` at build time, include it in the Lambda deployment bundle, and set the `BAML_LIBRARY_PATH` env var (or equivalent) so BAML finds it without downloading.

## Acceptance Criteria

- [ ] `build.sh` downloads the BAML native lib and includes it alongside the `bootstrap` binary
- [ ] Lambda environment variable tells BAML where to find the pre-bundled library
- [ ] Cold start no longer shows "Downloading libbaml_cffi..." in logs
- [ ] Init duration drops from ~1-2s to under 200ms (after BAML lib download eliminated)
- [ ] Local development still works (BAML handles its own lib locally)
- [ ] Verify with `curl` that the first request after deploy doesn't have the download delay
