## Context

Implement the HMW session store using Svelte 5 runes (`$state`, `$derived`). The store is the single source of truth for all workshop state: persona, problem context, analysis, candidates (with statuses), clipped IDs, and streaming state. The type definitions are already scaffolded in `session.ts`.

## Acceptance Criteria

- [ ] Session store uses Svelte 5 runes (`$state`) for reactive state
- [ ] Exposes methods: `setPersona()`, `setContext()`, `setAnalysis()`, `addCandidates()`, `updateCandidateStatus()`, `clipCandidate()`, `reset()`
- [ ] `clippedIds` derived from candidates with `CLIPPED` status
- [ ] `iterationCount` tracks how many refine cycles have been run
- [ ] `isStreaming` flag to prevent concurrent API calls
- [ ] Generates unique IDs for candidates (e.g., `crypto.randomUUID()`)
- [ ] Store is importable from any component via `$lib/stores/session`
