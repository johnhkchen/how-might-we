## Token Efficiency & Output Quality

LLM calls are the only real cost center. Current prompts are too verbose, some use Sonnet where Haiku suffices, and the interaction flow creates a bad UX where the analysis suggests a reframing that expansion then critiques.

### Track Breakdown

**Wave 1** (parallel — different files)
1. **T-008-01**: Replace single initial reframing with 3 scoped options the user picks from
2. **T-008-03**: Route cheap tasks (persona refinement) to Haiku

**Wave 2** (sequential — both touch all prompt files)
3. **T-008-02**: Constrain output verbosity across all prompts
4. **T-008-04**: Reduce variant generation from 6-8 to 4 per call

### DAG

```
T-008-01 ─┬→ T-008-02 → T-008-04
T-008-03 ─┘
```

T-008-01 is the biggest change (types + BAML + frontend). T-008-03 is small (client config + function annotations). Both complete before the verbosity/count changes touch the same files.
