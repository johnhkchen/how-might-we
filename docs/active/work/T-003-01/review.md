## Open concerns

### Node.js version warning

`eslint-visitor-keys@5.0.1` requires `node ^20.19.0 || ^22.13.0 || >=24` but the system runs `v23.3.0`. This produces a warning during install but doesn't cause runtime issues. Node 23.x is an odd-numbered release (not LTS). If this becomes a problem, the fix is to use Node 22 LTS.

### Tailwind ESM warning

Tailwind CSS 3.x's `load-config.js` emits an `ExperimentalWarning` when loading the ESM `tailwind.config.js` via `require()`. This is a known issue with Tailwind 3 + Node 22+. It's cosmetic and doesn't affect functionality. Upgrading to Tailwind 4 (when ready) would resolve it.

### Audit vulnerabilities

`npm audit` reports 7 vulnerabilities (2 low, 4 moderate, 1 high). These are in transitive dependencies and typical for a fresh SvelteKit project. None are in production-facing code paths (all deps are devDependencies). Should be addressed in a separate maintenance ticket if they persist.

### `apiFetch` eager initialization

`client.ts` evaluates `getApiFetch()` at module load time, capturing the mock/real decision once. This is correct for Vite's static env replacement (`import.meta.env.VITE_MOCK_API`) but means the decision can't change at runtime. Not a problem in practice, just a design note for future reference.

## Conclusion

The frontend dev environment is fully operational. All eight acceptance criteria pass. The four fixes were all configuration issues (ESM, dependency version, ESLint globals, Prettier ignores) — no architectural changes, no source code changes beyond one type fix in a test fixture.
